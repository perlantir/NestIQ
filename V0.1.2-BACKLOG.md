## PDF template — dynamic gain/loss color classes on refi comparison rows

Deferred from **Session 7.2b** (2026-04-21). Two rows in `pdf-refinance-with-masthead.html` (page 2 comparison matrix) have CSS classes (`gain`/`loss`) that were left hardcoded to the demo configuration:

1. **"vs current" delta row** (`rc-cell gain / gain / loss` for options A / B / C) — assumes options A + B lower the monthly payment and option C raises it (typical when C is a shorter-term product). If a borrower's quote set violates that pattern (e.g., option A raises payment), the green/red coloring will be wrong.
2. **"Lifetime net savings" row** (`rc-cell gain` on all three options) — assumes every option produces positive lifetime savings vs current. If any option has negative lifetime savings, the green coloring will be wrong.

Fix shape: either introduce 6 class tokens (`option_{a,b,c}_pi_delta_class`, `option_{a,b,c}_lifetime_savings_class`) or replace each row with a single sentinel (e.g., `<!--{{pi_delta_row}}-->`) and have the Swift builder emit sign-aware cell HTML. Sentinel approach keeps token count down but shifts more structure to Swift.

Low impact for v0.1.1 since demo-typical quote sets match hardcoded classes, but should be fixed before public App Store release.
